# Bootstrap Prompt

> 在新项目中运行 `project-bootstrap.sh` 后，将以下 prompt 发送给 AI，引导它根据项目信息填充 AGENTS.md 和 ai2ai 文档。
> 使用前替换 `{{...}}` 占位符。

---

## Prompt

```
我刚用 AI 协作方法论的 bootstrap 工具初始化了这个项目的文档结构。
请帮我根据项目信息填充 AGENTS.md 和 docs/ai2ai/status.md。

### 项目信息

- 项目名称：{{项目名称}}
- 一句话描述：{{项目做什么}}
- 技术栈：{{语言、框架、主要依赖}}
- 项目阶段：{{从零开始 / 已有代码基础 / 已上线维护}}

### 架构信息（如已确定）

- 分层结构：{{如 UI → API → Service → Store，或 前端 → BFF → 微服务}}
- 主要模块：{{列出核心模块和职责}}
- 关键约束：{{如依赖方向、不可修改的文件、特殊的构建流程}}

### 需要你做的事

1. **填充 AGENTS.md**：
   - 项目结构：根据实际目录结构填写
   - 架构概览：根据上面的架构信息填写
   - 全局规范：根据技术栈填写编码规范（语言版本、错误处理、日志、依赖管理等）
   - AI 决策边界：根据项目的关键约束，填充 Always do / Ask first / Never do
     - Always do 至少包含：遵循规范、更新 ai2ai、运行测试、按 task 执行
     - Ask first 至少包含：新增依赖、改 schema、改 API 签名、删文件、大范围重构
     - Never do 至少包含：覆盖 me2ai 文档，加上项目特有的禁止项

2. **填充 docs/ai2ai/status.md**：
   - 如果是新项目：当前阶段写"初始化"，模块就绪度全部标记为"❌ 未开始"
   - 如果已有代码：扫描项目目录，根据实际代码状态填写模块就绪度
   - 下一步建议：建议第一个迭代的目标（优先打通一条端到端链路）

3. **生成第一个 task 文件** `docs/ai2ai/tasks/001-xxx.md`：
   - 目标：项目的第一个迭代应该做什么（通常是搭建骨架、解决环境、打通流程）
   - 任务分解：3-8 个原子任务，每个有验证标准

完成后请列出你做了哪些填充，以及有哪些信息需要我补充或确认。
```

---

## 使用流程

1. 运行 `./project-bootstrap.sh ~/projects/my-new-app`
2. 在 IDE 中打开项目
3. 复制上面的 prompt，替换 `{{...}}` 占位符
4. 发送给 AI
5. 审核 AI 生成的内容，确认或调整
6. 开始第一个迭代
