# 生成验收 Checklist

根据当前迭代的 task 文件，生成或更新 `docs/ai2ai/checklist.md`。

---

## 执行步骤

### 1. 读取当前迭代信息

从 `docs/ai2ai/status.md` 找到当前活跃迭代编号，读取对应的 `docs/ai2ai/tasks/NNN-xxx.md`。

### 2. 读取上一迭代 checklist（回归继承）

读取 `docs/ai2ai/checklist.md` 现有内容，提取已通过的测试项作为回归测试。
同时读取 `docs/ai2ai/test-suite.md` 中的全局用例（去重合并）。

### 3. 生成新 checklist

将 task 文件的"验收标准"展开为具体操作步骤 + 预期结果，格式如下：

```markdown
# 测试 Checklist

> AI 生成和维护，人审核和勾选。
> 当前迭代：NNN-xxx

## 本迭代新增
- [ ] [测试项描述]：[操作步骤] → [预期结果]

## 回归测试
- [ ] [测试项描述]：[操作步骤] → [预期结果]
```

生成规则：
- **本迭代新增**：从 task 验收标准展开，每项包含具体操作步骤和预期结果
- **回归测试**：继承上一迭代通过的测试项 + test-suite.md 全局用例（去重）
- 上一迭代失败项：本迭代修复了 → 移到"本迭代新增"；未修复 → 保留并标注 `⚠️ 上次失败`

### 4. 写入 checklist.md

覆盖写入 `docs/ai2ai/checklist.md`，输出生成摘要（新增 N 项，回归 M 项）。

---

## 失败项处理（用户测试后）

用户标记失败时的格式：

```markdown
- ❌ [测试项描述]
  > 现象：...
  > 备注：...
```

收到失败项后：

| 类型 | 判断 | 处理 |
|------|------|------|
| 当轮修复 | 属于本迭代验收标准范围 | 修复代码 → 将该项改回 `[ ]` → 等用户重新测试 |
| 下轮修复 | 超出本迭代范围 | 标注 `→ 转入下一迭代`，下一迭代自动生成修复任务 |

安全护栏：同一项连续修复 3 次仍未通过 → 转为下轮修复并报告。
